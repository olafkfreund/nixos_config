# AI Tooling Customization Plan — Claude Code + Google Antigravity

> Created: 2026-06-11
> Scope: Make both AI coding tools declarative, DRY, and tuned to this multi-host NixOS workflow.
> Status: Proposed (research complete; nothing implemented yet)

This plan is the output of researching the customization surfaces of **Claude Code**
and **Google Antigravity** (official docs, blogs, community repos) and cross-referencing
them against the *actual* state of this machine and repo.

---

## TL;DR — the 5 highest-leverage moves

| # | Move | Why it matters | Effort |
|---|------|----------------|--------|
| 1 | **One `AGENTS.md` for both tools** | Claude Code *and* Antigravity both read `AGENTS.md`. Today you maintain Claude context in `~/.claude/CLAUDE.md` + repo `CLAUDE.md`, and Antigravity has *nothing*. Collapse the shared essence into one declaratively-managed `AGENTS.md`. | S |
| 2 | **Make Claude Code config declarative** | `~/.claude/settings.json` and `~/.claude/CLAUDE.md` are currently **mutable, hand-edited files** — they drift and aren't reproducible across p620/razer. Bring them into home-manager (mutable-merge pattern, same as the VS Code fix). | M |
| 3 | **Generate Antigravity config from Nix** | `~/.gemini/config/mcp_config.json`, `GEMINI.md`, `.agent/workflows/*` are all plain files → generate from the **same Nix MCP/rules source** you already maintain for Claude Code. Antigravity is installed but unconfigured. | M |
| 4 | **Auto-format + lint hook** | A `PostToolUse` hook running `alejandra`/`statix` on edited `.nix` files removes a whole manual step before every commit. | S |
| 5 | **Curate the agent/skill sprawl** | 62 agents + ~30 skills + 18 commands, mostly generic. Trim/namespace so autocomplete surfaces *your* tools (`/nix-*`, `/media:*`) not 40 irrelevant ones. | S |

## Current state (audit)

### Claude Code — already rich, partially declarative

- **Agents:** 62 in `~/.claude/agents/` (mostly the generic "awesome-agents" pack — sales, minecraft, ios, etc.)
- **Commands:** 18 in `~/.claude/commands/` (some symlinked→nix store, some mutable)
- **Skills:** ~30 in `~/.claude/skills/` (mix of global AbsolutelySkilled + project)
- **Hooks:** 14 events wired in `settings.json`; PARR enforced via `~/.claude/hooks/parr-watch.sh`
- **Repo modules:** `home/development/claude-code-lsp.nix`, `claude-code-mcp.nix`, `home/shell/claude-integration.nix`, `home/default.nix` (`programs.claude-code`), `modules/programs/default.nix`
- ⚠️ **Gaps:** `settings.json` + `~/.claude/CLAUDE.md` are **mutable** (drift risk, not reproduced on other hosts). MCP config in a 268 KB mutable `~/.claude.json`. No `output-styles/`. No auto-format hook. No custom `statusLine` script in repo.

### Antigravity — installed, not configured

- **Install:** flake input `antigravity-nix`, `modules/ai/antigravity-cli.nix`, `pkgs/antigravity-cli`, `pkgs/google-antigravity-py`; enabled on p620/razer/p510 via features.
- **Config dir `~/.gemini/` exists** (`antigravity/`, `antigravity-cli/`, `antigravity-ide/`, `config/`, `agents/`, `commands/`) — all hand-managed / agent-managed, **none generated by Nix**.
- ⚠️ **Gaps:** no `AGENTS.md`/`GEMINI.md`, no declaratively-managed workflows, no Nix-generated `~/.gemini/config/mcp_config.json`.
- ⚠️ **Hard limit (not your fault):** Antigravity is **OAuth-only — no BYOK/API-key**, so you *cannot* declaratively inject keys or run it fully offline today. Track upstream (`antigravity-cli` issue #78).

## Customization surface reference (condensed)

### Claude Code — what's actually configurable

- **`settings.json`** (`~/.claude/` user, `.claude/` project): `model`, `permissions` (allow/ask/deny, Deny wins), `env`, `hooks`, `statusLine`, `outputStyle`, `theme`, plugins/marketplaces, MCP enable flags.
- **Hooks** (verified events incl. `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `UserPromptSubmit`, `Stop`, `SessionEnd`, `SubagentStart/Stop`, `Notification`, `PermissionRequest/Denied`, `PreCompact/PostCompact`): shell command reads JSON on stdin, exit code gates the action. You already use these for PARR.
- **Slash commands / skills** (`~/.claude/commands/`, `.claude/skills/`, `SKILL.md` frontmatter + args).
- **Subagents** (`.claude/agents/*.md`, frontmatter: name/description/tools/model; optional worktree isolation).
- **Output styles** (`~/.claude/output-styles/*.md`) — replace the system prompt for a role.
- **Memory** — `CLAUDE.md` hierarchy with `@./path` imports; auto-memory dir.
- **MCP** — project `.mcp.json` / `~/.claude.json`.
- **Headless** — `claude -p "…" --allowedTools …` for cron/CI.
- ⚠️ *Caveat:* some "settings keys" floating around blogs are speculative; stick to keys you can confirm in `claude config`/your own working `settings.json`.

### Antigravity — what's actually configurable (early/mid-2026, churning)

- **Rules/memory:** `~/.gemini/GEMINI.md` (global, highest), `~/.gemini/AGENTS.md` (cross-tool), per-repo `GEMINI.md` / `AGENTS.md` / `.agent/rules/*.md`. Precedence: System → GEMINI.md → AGENTS.md → `.agent/rules/` → defaults.
- **Workflows:** `<repo>/.agent/workflows/*.md` (`/name`, 12k-char cap) and global `~/.gemini/antigravity/global_workflows/*.md`.
- **Skills (AG 2.0):** `~/.gemini/skills/<name>/SKILL.md` — *same `SKILL.md` format as Claude Code* → portable.
- **MCP:** `~/.gemini/config/mcp_config.json` (`mcpServers` block ≈ Claude Code's; `url`→`serverUrl`, drop `timeout`).
- **Permissions:** 4 autonomy profiles + allow/ask/deny (Deny>Ask>Allow). *(Allowlist is buggy in preview — don't over-rely.)*
- **Models:** multi-vendor, per-agent (Gemini 3.x / Claude Sonnet-Opus / GPT-OSS) — a real differentiator vs Claude Code's single-vendor.
- **VS Code fork:** settings/keybindings/extensions (Open VSX) import; manage via `programs.antigravity` HM module or `home.file`.

## The plan — phased

### Phase 1 — DRY foundation (do first) `S–M`

**Goal:** one source of shared AI context, both tools declarative.

1. **Author `AGENTS.md` at the repo root** distilled from `CLAUDE.md` (architecture, anti-patterns pointer, host map, deploy commands, PARR summary). Keep it tool-neutral.
   - Claude Code already reads `AGENTS.md`; Antigravity reads it too. Keep tool-specific bits in `CLAUDE.md` (Claude) and a thin `GEMINI.md` (Antigravity) that both `@import`/reference `AGENTS.md`.
2. **Bring `~/.claude/settings.json` into home-manager** as a *mutable-merge* file (reuse the exact pattern just built for VS Code `settings.json`: render declarative keys to a store file, jq-merge into a mutable file on activation so Claude's own writes survive). Declarative keys to pin: `model`, `permissions.allow` (your `just *`, `nix build`, repo reads), `hooks` (PARR + new format hook), `statusLine`, `outputStyle`.
3. **Bring `~/.claude/CLAUDE.md` (global) into home-manager** the same way, or make it a thin file that `@import`s the repo `AGENTS.md`.

**Verify:** `settings.json`/`CLAUDE.md`/`AGENTS.md` reproduce identically on p620 + razer after a switch; Claude still starts clean.

### Phase 2 — Antigravity declarative config `M`

**Goal:** Antigravity gets the same context + tools, generated from Nix.

1. **New module `home/development/antigravity-config.nix`:**
   - `xdg.configFile`/`home.file` → `~/.gemini/AGENTS.md` = symlink/copy of the repo `AGENTS.md` (single source).
   - Generate `~/.gemini/config/mcp_config.json` from the **same Nix attrset** that feeds Claude Code's MCP (transform `url`→`serverUrl`, drop `timeout`) via `(pkgs.formats.json {}).generate`.
   - Port 2–3 of your most-used `/nix-*` flows to `~/.gemini/antigravity/global_workflows/*.md` (e.g. a deploy walkthrough, a validate-before-commit recipe).
2. **Reuse SKILL.md portability:** symlink a couple of your `~/.claude/skills/<x>/SKILL.md` into `~/.gemini/skills/<x>/` where format-compatible.
3. **Document the BYOK limitation** in the module header so future-you knows keys are interactive/OAuth-only.

**Verify:** `agy`/IDE picks up `AGENTS.md` + MCP servers; workflows show under `/`.

### Phase 3 — Workflow ergonomics `S`

1. **Auto-format hook (Claude Code):** `PostToolUse` matcher `Edit(*.nix)` → run `alejandra` (or `nixpkgs-fmt`) + `statix` on the edited file. Mirrors your `just format` but automatic.
2. **Custom `statusLine` script** (declarative): show host, git branch, dirty state, model, effort — handy in tmux across hosts.
3. **Permission allowlist** to kill prompt fatigue: pre-allow `Bash(just *)`, `Bash(nix build .#*)`, `Bash(nix flake check)`, repo `Read(...)`. (There's also a `/fewer-permission-prompts` skill that scans transcripts and proposes an allowlist — run it once.)
4. **Output styles** for recurring roles: `nix-architect.md` (module/perf focus), `security-auditor.md`. Stored in `~/.claude/output-styles/`, declaratively.

### Phase 4 — Curate the sprawl `S`

1. **Trim agents:** of 62, keep the ~15 relevant (nixos-pro, code-reviewer, security-auditor, debugger, deployment-engineer, devops-troubleshooter, docs-architect, the language-pros you use). Archive the rest so they stop diluting autocomplete/selection.
2. **Namespace commands/skills** consistently (`/nix-*`, `/media:*`) and drop dead ones.
3. **Optionally** use `claudeMdExcludes`/skill visibility settings to keep the surface lean.

### Phase 5 — Automation (optional, higher effort) `M–L`

1. **Scheduled headless audits:** `claude -p "/nix-security"` / `"/check-updates"` via a systemd timer or `/schedule`, emitting to a log or GitHub issue.
2. **PR-time review hook:** wire `/code-review` into your existing GitHub flow.
3. **Antigravity browser-verification** for any web UI you maintain (its built-in browser subagent is its standout feature; Claude Code needs the Chrome MCP for parity — which you already have).

## What I deliberately did *not* recommend

- **Chasing Antigravity BYOK/offline** — not supported yet; revisit when upstream lands it.
- **Speculative `settings.json` keys** from blogs — only pin keys you can confirm work on your install.
- **A second memory system** — `AGENTS.md` + thin tool files is enough; don't fragment context.
